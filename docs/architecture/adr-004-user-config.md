# ADR-004: 配置与仓库发现——用户级 TOML 显式声明

## Status

Accepted

## Context

**问题**：应用需要知道"关注哪些 git 仓库"，以及时钟、天气等参数。仓库列表如何确定？配置放哪里？

**约束**：
- 用户希望由配置显式声明关注的仓库，而非应用自作主张。
- 配置应放在 home 目录下常规的、用户级的配置目录中。
- 两端（Linux/macOS）应使用**同一份配置格式**，由各端按同一规格加载与校验。

## Options Considered

| 方案 | Pros | Cons | 复杂度 | 何时合适 |
|------|------|------|--------|----------|
| A. 用户级配置文件显式列出仓库 | 精确可控；无意外扫描；可版本化（chezmoi 自管） | 新增仓库需改配置 | 低 | 仓库集合相对稳定时 |
| B. 给根目录自动递归扫描 `.git` | 零维护、自动发现 | 可能扫到不关心的仓库；大目录慢；行为不可控 | 中 | 仓库很多且都想看时 |

## Decision

**选择方案 A 为主**：使用用户级 TOML 配置文件显式声明关注的仓库及其它参数。
- 路径遵循 XDG 规范：`$XDG_CONFIG_HOME/w_dashboard/config.toml`，回退到 `~/.config/w_dashboard/config.toml`（macOS 同样采用 `~/.config/w_dashboard/` 以与 Linux 一致并便于 chezmoi 跨机管理）。
- **可选增强**：允许配置 `scan_roots`（扫描根目录）作为补充，但默认不启用；显式 `repos` 列表始终是主路径。

配置内容涵盖：关注的仓库列表、是否启用 chezmoi、时钟（多个 IANA 时区）、天气（Open-Meteo 经纬度/地点）、刷新间隔等。详见 SDD 的配置 schema。

## Rationale

1. 满足"由配置显式声明、用户级"的明确要求。
2. 显式列表行为可预测，避免递归扫描扫到无关仓库或带来性能问题。
3. 配置文件位于 `~/.config/`，可直接交给 chezmoi 跨机管理——这与本系统监控 chezmoi 的主题自洽。
4. 两端共享同一 TOML 格式与同一份解析规格（含字段、默认值、校验规则），保证一致。

## Trade-offs

- 新增/移除关注的仓库需要手动编辑配置（可被 `scan_roots` 可选缓解）。

## Consequences

- **Positive**：行为精确可控；配置可版本化、可跨机同步；解析规则集中在规格、两端一致实现。
- **Negative**：仓库集合变化时需维护配置。
- **Mitigation**：提供清晰的默认配置模板与字段文档；保留可选 `scan_roots` 作为自动发现补充。

## Revisit Trigger

- 手动维护仓库列表成为明显负担，需要把自动扫描提升为默认行为。
